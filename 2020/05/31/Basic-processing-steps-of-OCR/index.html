Basic processing steps of OCR

Parity testing, which means to compare the execution result of all programs from old/new systems, is usually an important phase in mainframe modernization project. And recently we are trying to upgrade it to a OCR based automated, standard process.

So letâ€™s check the basic processing steps of OCR and find out most notable points for improving the OCR accuracy rate.

  1. Image importing

    Import the entire document as an image with a scanner or camera

  2. Layout check

    Analyse the image structure and divide it into character, border line, photo, etc. And check out the part which should be read as characters

  3. Recognition (cutting out characters)

    Cut out the character units from character string.

  4. Recognition (character recognition)

    Recognize characters by their characteristics.

  5. Format output

    Output text data to a file

The most important step is to recognize the characters after cutting them out from character string(4), and especially for Chinese characters, the cutting out step(3) itself would be considered as another key factor to increase accuracy rate(refer to the negative case below for reason). Furthermore, it also became possible to perform much higher accuracy rate by introducing AI in step 3/4 such as deep learning and reinforcement learning. We speculate achieving 99% accuracy rate should be a standard line or requirement to implement the OCR tool to actual project.

negative case

Another reason we consider to use OCR in parity testing is that the object images in mainframe modernization project are always similar, actually there are not so many layout patterns especially in mainframe part.
