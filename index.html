Screenless Online Automation Testing

Now introducing automation tools for online testing process is quite usual, automated screen operation would save more than 90% of the testing time which cost by the manual action such as inputting, clicking and taking screenshot. But is it possible to use that tools on a remote server without screen, or the server that we are not free to use its screen cause it run several important programs all the time?

The answer is YES. As an example I’ll show an automation online testing program that written by Python and working with Selenium, it will run in background and use Chrome to open a google homepage, input searching keyword, click “search” button and take all screenshots automatically. also, it should be pointed out that the key of “screenless” setting here is the option argument “–headless”.

Concurrent session control by Spring Security

We all have to face the problems of concurrent login control while building a new website, and basically there are 2 ways to solve that, DB and session management. Unless system ensure all the login and logout events are collected, session management would be the best solution for this kind of problems, and Spring Security provide several features that can help us to implement it.

So just consider the situation that a user has already logged in the system but still attempts to login again from different applications or devices, for which we might stop the current login and show an alert like “you’ve logged from xxx”(No.1), or allow the current login and disconnect the previous one (No.2).
